# 五子棋游戏使用说明

## 游戏简介

这是一款功能完整的五子棋游戏，支持单人模式（与AI对战）、本地双人对战和局域网对战三种游戏模式。游戏界面简洁美观，支持响应式布局，可在不同设备上流畅运行。

## 快速开始

### 启动游戏

**方式一：直接打开（单人/本地双人模式）**
1. 直接在浏览器中打开 `index.html` 文件
2. 推荐使用 Chrome、Firefox、Edge 等现代浏览器

**方式二：启动服务器（局域网对战模式）**
```bash
# 安装依赖
npm install

# 启动服务器
npm start
```
3. 服务器启动后访问 http://localhost:3000

### 游戏界面说明

游戏界面分为三个主要区域：

- **左侧边栏**：游戏模式选择、AI难度设置、局域网设置、棋子样式选择
- **中央区域**：棋盘、游戏状态显示、控制按钮
- **右侧边栏**：游戏记录、游戏统计

## 游戏模式

### 1. 单人模式（人机对战）

- 点击左侧"单人模式"按钮进入
- 玩家执黑先行，AI执白
- 可选择三种AI难度：
  - **简单**：AI随机落子，适合新手练习
  - **中等**：AI具备基本策略，有一定挑战性
  - **困难**：AI使用高级算法，需要认真思考才能获胜

### 2. 本地双人模式

- 点击左侧"本地双人"按钮进入
- 两位玩家轮流在同一设备上下棋
- 黑方先行，双方交替落子

### 3. 局域网对战模式

- 点击左侧"局域网对战"按钮进入
- 需要先启动 Node.js 服务器

**创建房间：**
1. 输入服务器地址（如 http://192.168.1.100:3000）
2. 点击"连接服务器"
3. 点击"创建房间"
4. 将生成的6位房间码告诉对手
5. 等待对手加入

**加入房间：**
1. 输入服务器地址并连接
2. 点击"加入房间"
3. 输入房主提供的房间码
4. 点击"加入"

**注意事项：**
- 房间创建者执黑先行
- 支持悔棋请求和重新开始请求（需对方同意）
- 局域网对战需要双方连接同一服务器

## 游戏操作

### 落子

- **鼠标操作**：点击棋盘交叉点落子
- **触摸操作**：在移动设备上点击棋盘落子
- 落子后不可更改，请谨慎思考

### 游戏控制

- **悔棋**：撤销上一步落子
  - 单人模式：同时撤销玩家和AI的落子
  - 双人模式：撤销最后一步
  - 局域网模式：需对方同意

- **重新开始**：清空棋盘，重新开始游戏

- **暂停/继续**：暂停或继续游戏计时

### 棋子样式

提供两种棋子样式可供选择：

- **经典样式**：传统黑白棋子外观
- **现代样式**：简约现代风格

点击左侧棋子样式预览即可切换。

## 游戏规则

1. 黑方先行，双方轮流落子
2. 先将五颗棋子连成一线（横、竖、斜均可）的一方获胜
3. 棋盘填满且无人获胜则为平局

## 游戏功能

### 计时功能

- 分别记录黑方和白方的用时
- 游戏开始自动计时
- 暂停时停止计时

### 游戏记录

- 实时显示落子历史
- 格式：序号. 玩家 坐标（如：1. 黑 A8）
- 坐标说明：字母代表列（A-O），数字代表行（1-15）

### 游戏统计

- 记录总局数、黑方胜局、白方胜局
- 数据保存在浏览器本地存储中
- 清除浏览器数据会重置统计

### 技术栈

- **前端**：HTML5 Canvas + 原生 JavaScript
- **后端**：Node.js + Express + Socket.io
- **通信**：WebSocket 实时双向通信

### AI算法

AI使用Minimax算法配合Alpha-Beta剪枝：

- 评估函数考虑连子数量、开放端数、封堵情况
- 不同难度对应不同搜索深度
- 困难模式会进行更深入的局势分析

## 常见问题

### Q: 局域网对战无法连接？

A: 请确保：
- 服务器已正确启动
- 双方使用正确的服务器地址
- 防火墙允许 3000 端口访问

### Q: 如何在局域网中使用？

A: 
1. 在一台电脑上启动服务器：`npm start`
2. 查看本机IP地址（Windows: `ipconfig`，Mac/Linux: `ifconfig`）
3. 其他设备访问 `http://本机IP:3000`

### Q: 游戏数据丢失？

A: 游戏数据保存在浏览器本地存储中，清除浏览器数据会导致数据丢失。
